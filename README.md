# batch-article-summary-skill

一个把多篇文章批量整理成高可读性 HTML 页面的小型 Skill。

它适合处理“阅读列表”式的文章集合：内容很多、主题不一定统一，但你希望一次性整理出来，方便快速回看、筛选和分享。

## 它解决什么问题

很多文章批量总结工具都有两个常见问题：

- 只会吐出一大段文字，不方便扫读
- 喜欢强行给整批文章找一个“共同主题”，但真实世界里的阅读列表往往就是东一篇西一篇

这个 Skill 的目标更务实：

- 按原顺序整理多篇文章
- 为每篇文章产出结构化摘要
- 最终输出一个可直接打开的 HTML 页面
- 页面更像“阅读清单整理页”，不是“硬凑专题页”

## 适用场景

- 微信文章合集
- 资讯阅读清单
- 抓取后的文章 JSON
- 一次性整理十几篇甚至更多文章
- 想快速判断“哪些值得细读，哪些可以跳过”

## 输入格式

默认支持两种结构：

### 1. 纯数组

```json
[
  {
    "title": "文章标题",
    "url": "https://example.com/article",
    "content": "文章正文"
  }
]
```

### 2. 包装对象

```json
{
  "articles": [
    {
      "title": "文章标题",
      "url": "https://example.com/article",
      "content": "文章正文"
    }
  ]
}
```

每篇文章至少包含：

- `title`
- `url`
- `content`

## 输出效果

Skill 会生成一个单文件 HTML 页面，默认落到：

```text
content/fetch_content/
```

输出页面包含这些特性：

- 顶部是阅读列表式页头，不强行总结整批文章
- 左侧或顶部有目录导航
- 每篇文章一张卡片
- 标题可点击跳转原文
- 自动区分：
  - 普通文章
  - 软文
  - 内容不足无法总结的文章

## 单篇文章结构

普通文章默认整理为四块：

1. 推荐指数
2. 这篇讲了啥
3. 关键细节
4. 深度解读

其中有几个明确取向：

- `这篇讲了啥`：优先短、清楚、先给结论
- `关键细节`：不是空洞摘要，而是尽量保留人物、动作、数字、冲突和反差
- `深度解读`：往下挖一层，但不脱离原文，不硬灌鸡汤

## 风格特点

这个 Skill 不是为了把总结写得像散文，而是为了让页面更适合阅读。

所以它会优先追求：

- 易扫读
- 分段明显
- 重点句能站出来
- 阅读节奏像“高质量笔记”

而不是：

- 一整块密密麻麻的大段文字
- 对每篇都写成长篇评论
- 对整批文章硬造一个共同主题

## 软文识别规则

它不会因为文章里出现品牌、活动、平台链接就直接判软文。

只有在以下情况更明显时，才会判为软文：

- 推广是全文主目的
- 正文大部分篇幅都在为转化服务
- 去掉推广对象后，几乎没有独立信息价值

另外，这类文章默认仍按普通文章处理：

- 活动体验
- 参会随笔
- 观展记录
- 线下见闻

前提是正文仍有真实见闻、人物观察、现场细节或独立观点。

## 关键文件

- `SKILL.md`
  - Skill 的触发规则、输入约定和执行流程
- `references/prompt-template.md`
  - 实际总结时使用的提示词模板
- `assets/report-template.html`
  - 页面骨架和样式模板
- `scripts/next_output_path.sh`
  - 生成输出文件路径

## 适合怎样使用

如果你在本地工作流里已经有文章抓取结果，只需要把结果组织成文章数组，再让 Skill 去做最后一公里：

- 批量总结
- 结构化整理
- 生成为 HTML
- 直接打开浏览

它更像一个“阅读清单整理器”，不是一个“通用网页爬虫”。

## 备注

当前版本特别强调：

- 不强行为整批文章制造统一主题
- 页面更像阅读列表，而不是杂志专题封面
- 单篇内容要有抓手，不要写成抽象空话
